# 吃货遗愿清单 — PWA 设计方案

> 糖糖（2015.7.24）："名字想好叫 吃货遗愿清单哈哈哈哈"
> 概念：基于"死亡视角"的美食体验 List —— 在 App 里列出这一生要吃的所有东西，完成打卡。

---

## 一、产品定位

**不关乎效率，关乎在有限的生命里，把想要的东西列出来，然后一个一个吃掉。**

这不是餐饮点评工具，不是大众点评收藏夹。它的核心情感语境是：

- 你终有一死
- 在死之前，你想吃什么？
- 列出来
- 吃掉
- 每吃掉一个，你离"完成"近了一步，也离死亡近了一步

这是一种 **memento mori（死亡提醒）** 式的产品哲学——不是悲观，而是通过死亡的紧迫感来给日常的吃赋予仪式感和意义。

---

## 二、目标用户与应用场景

### 用户画像
- 热爱食物、有探索欲的人
- 喜欢列清单、打卡、有"完成欲"的人（糖糖自己就是）
- 对"有限生命"有自觉意识的年轻人

### 核心场景
1. **发现美食** → 打开 App，记录下来（"这辈子一定要吃一次北海道海胆饭"）
2. **计划去吃** → 看到有一条"还没吃"，触发周末出行计划
3. **打卡完成** → 吃掉后拍照记录，打上 ✓，那一刻有"又完成了一件"的满足感
4. **回顾** → 凌晨两点，翻看自己吃完的东西，像翻一本和自己和解的书

#### 功能点：
1. **添加心愿**
   - 美食名称（必填）
   - 分类标签：🍣日料 / 🍜面食 / 🥮点心 / 🍲火锅 / 🥩烤肉 / 🍕西餐 / 🍺饮品 / 📍地方小吃 / 自定义
   - 想去的地方/餐厅（选填）
   - 为什么想吃/备注（选填）——让每一条都有"故事感"
   - 优先级：⭐ 此生必吃 / 很想吃 / 有机会就吃

2. **打卡完成**
   - 点击"吃到了！"→ 弹出记录页面
   - 拍照（或从相册选图）
   - 日期自动记录
   - 一条笔记：味道怎么样？值得推荐吗？
   - 打卡后自动归档到"已完成"

3. **清单浏览**
   - 按状态筛选：待完成 / 已完成
   - 按分类筛选
   - 按优先级排列
   - 搜索

4. **仪表盘**
   - 一个简洁的统计面板
   - 已完成数 vs 待完成数
   - "如果生命只剩 X 天，你每天需要吃 Y 个才能吃完"——糖糖视角的核心功能

5. 🔥 **懒人一键添加**（内置美食数据库）
   - App 内置一份精心整理的美食数据库（约 50-80 条经典美食）
   - 用户点击"懒人一键添加" → 预览数据库内容 → 勾选想要的 → 确认
   - 系统自动将选中的条目批量导入到个人清单
   - **智能去重**：如果用户已手动添加过同名美食，自动跳过不重复添加
   - **分类预览**：按菜系/地域分组展示，方便浏览和选择

### 第二层：情感功能（糖糖式的细节）

5. **"遗愿倒计时"**
   - 用户可选填一个"假设日期"（不一定要是真的）
   - 自动计算："距离X还有N天，你还有M个心愿没完成"
   - 这是一种游戏化的存在主义——**和死亡比赛吃东西**

6. **心愿随机推荐**
   - 按钮："今天不知道吃什么？"
   - 随机抽取一条未完成的心愿推荐
   - 糖糖会说"完成游戏任务一样"——这个按钮就是 quest log

7. **年度总结**
   - 每年生成一次："这一年你吃了 X 样心愿美食"
   - 分享卡片（可截图分享到微信）

### 第三层：未来可扩展

8. **地图视图** —— 所有心愿按地理位置标记
9. **好友共享** —— 看到朋友的心愿清单
10. **餐厅推荐** —— 基于心愿类型推荐附近餐厅

## 三-B、内置美食数据库设计

### 数据库定位
这不是"权威美食指南"，而是一份**有温度的美食清单**——收录的是"一个人一生至少该吃一次"的经典食物。每条条目都带一句"为什么值得吃"，让糖糖看到的不是冷冰冰的数据库，而是一份有人情味的邀请。

### 数据库规模
- **MVP 版本**：50 条经典美食
- **可扩展**：后续可通过更新代码追加更多条目
- **结构**：每条 = 名称 + 分类 + 推荐地点 + 一句话理由 + 默认优先级

### 关键交互细节
- **默认全选**：50 道全部勾上，用户可以直接点"一键导入"
- **分组折叠**：按菜系/地域分组，每组的条目数显示在标题旁
- **智能去重**：点"一键导入"时，自动检测用户清单中已有的同名条目，跳过不重复，并在按钮上显示实际导入数量
- **可逆操作**：导入后，每一条都像手动添加的一样独立存在——可以单独删除、修改、打卡
- **首次引导**：用户第一次打开 App、清单为空时，自动弹出懒人添加引导

### 技术实现要点
- 内置数据库作为 **JS 常量数组**硬编码在 `app.js` 中（50条约 5KB，几乎不增加体积）
- 导入时逐条写入 IndexedDB，每条的 `source` 字段标记为 `"builtin"` 以区别于手动创建
- 去重逻辑：按 `名称 + 分类` 做 key 匹配（避免"兰州牛肉面"和"兰州拉面"被认为是同一条的情况——保守去重，宁可少量重复也不错杀）

## 四、技术架构（PWA）

### 技术选型

```
技术栈：
├── 前端：纯 HTML + CSS + JavaScript（零依赖，极致轻量）
├── 存储：IndexedDB（浏览器本地数据库，容量远超 localStorage）
├── 离线：Service Worker（PWA 核心，缓存所有资源）
├── 推送：Web Push API（iOS 16.4+ 支持，提醒"该去吃了"）
├── 图片：FileReader + Canvas（压缩后存入 IndexedDB）
└── 部署：任意静态托管（GitHub Pages / Vercel / 腾讯云静态托管）
```

### 为什么不用框架
- 这个 App 的交互模式简单：列表、表单、大图
- PWA 体积应该尽可能小（< 100KB）
- 对方不需要安装任何依赖
- 纯 HTML/CSS/JS 意味着——你只需要分享一个 HTML 文件，就能跑起来

### 数据流

```
用户操作 → JS 逻辑 → IndexedDB（本地存储）
                          ↓
                    Service Worker 缓存
                          ↓
                    离线也能用
```

关键设计：
- **所有数据存本地**：不依赖服务器、不需要登录、不需要网络
- **照片压缩后存储**：每张图限制在 200KB 以内
- **数据导出**：JSON 格式导出 → 可以通过微信/邮件发送备份
- **数据导入**：从 JSON 恢复 → 换手机不丢数据

### PWA 关键文件

```

│   ├── icon-192.png        # 192x192 图标
│   └── icon-512.png        # 512x512 图标

```

## 五、视觉设计方向

### 命名与图标
- **App 名**：吃货遗愿清单（尊重糖糖的命名）
- **英文名**：Foodie's Bucket List
- **Slogan**：吃完再说
- **图标**：一副筷子 + 一个沙漏，或一个碗 + 倒计时数字

### 配色方案

```
主题色：    #D4514E  （温暖的暗红——像食物，也像提醒）
背景色：    #FAFAFA  （米白——温和，不刺眼）
卡片色：    #FFFFFF  （白色卡片，干净）
已完成：    #4A9E6D  （绿色——完成感）
未完成：    #E8D5B7  （淡米色——等待）
警示/强调： #E74C3C  （红色——时间/死亡提醒）
文字主色：  #2C2C2C  （深灰——安静）
文字辅色：  #8C8C8C  （浅灰——次要信息）
```

### 字体
- 中文字体：系统默认（苹方 / 微软雅黑）
- 数字：等宽字体（用于倒计时数字）

### 卡片设计
- 每一条心愿是一张卡片
- 已完成的心愿：照片占左侧，文字在右侧，底部一条绿色下划线
- 未完成的心愿：大图标占左侧，文字在右侧，底部一条虚线
- 有照片的打卡记录：放大照片，模糊背景

### 启动画面设计
- 深色背景 + 中间一个碗的剪影
- 下方一行小字："吃完再说"
- 简单、安静、有分量

## 六、交互流程设计

### 首次打开体验
```
打开链接 → 看到启动画面（2秒）
         → 进入主页面（清单为空）
         → 引导卡片："你还有多少东西没吃？"
```

## 七、分享机制 —— 如何"发给她安装"

### 一句话描述流程
> 你部署到服务器 → 拿到一个链接 → 发给她 → 她 Safari 打开 → 点"添加到主屏幕"→ 完成

### 具体步骤（你的角度）
1. 我把 PWA 源码写好，所有文件在一个文件夹里
2. 部署到 GitHub Pages（免费）或其他静态托管
3. 你会得到一个类似 `https://xxx.github.io/chihuo-bucketlist/` 的链接
4. 把这个链接通过微信发给她
5. 告诉她："用 Safari 打开 → 点底部分享按钮 → 添加到主屏幕"

### 具体步骤（她的角度）
1. 微信收到链接 → 点开 → 右上角"..."→ "在 Safari 中打开"
2. Safari 打开后 → 底部中间"分享"按钮（方框+箭头）
3. 向下滑 → "添加到主屏幕"
4. 主屏幕上出现"吃货遗愿清单"图标
5. 点击图标 → 全屏打开，像一个真正的 App

### 后续更新
- 你更新代码 → 重新部署 → 她下次打开 App 时自动更新（Service Worker 机制）
- 不需要她重新下载或重新添加

## 八、可行性评估

| 维度 | 评估 |
|------|------|
| 技术可行性 | ✅ 极高，PWA 是成熟技术 |
| iOS 兼容性 | ✅ Safari 支持 PWA 添加到主屏幕（iOS 11.3+） |
| 离线使用 | ✅ Service Worker + IndexedDB |
| 推送通知 | ⚠️ iOS 16.4+ 才支持 Web Push，但核心功能不需要推送 |
| 拍照 | ✅ 使用 `<input type="file" capture="camera">` |
| 数据持久性 | ✅ IndexedDB 不会像 localStorage 那样被系统清理 |
| 微信内使用 | ⚠️ 微信内置浏览器部分限制，建议引导至 Safari |
| 发布成本 | ✅ 零成本（GitHub Pages 免费） |
| 隐私安全 | ✅ 数据全在本地，不上传服务器 |

## 九、与糖糖原概念的对应

| 糖糖说的 | 设计中的体现 |
|---------|------------|
| "吃货遗愿清单" | 产品名，一字不改 |
| "一生想要吃的所有东西" | 手动无限添加 + 内置 50 道一键导入，按优先级分类 |

| "打卡" | 拍照 + 日期 + 笔记的完整打卡流程 |
| "现在是做不成的" | 今天把它做成，11年后还给她 |
